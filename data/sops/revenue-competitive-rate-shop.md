# Competitive Rate Shop Procedure

**Property:** Salinas Point Inn, Central California Coast
**Department:** Revenue Management
**Document Owner:** General Manager
**Version:** 1.0

## Purpose

The competitive rate shop is how the property sees itself in context. Without regular comp set data, rate decisions are made in isolation — the property may be leaving money on the table or underpricing against a softening market without realizing it. This procedure defines how the General Manager at Salinas Point Inn conducts the weekly competitive rate shop, interprets the data, and integrates findings into pricing decisions.

## Scope

Applies to the General Manager (holding the Revenue Manager function). Covers the defined comp set, rate shop methodology, competitive response decisions, and comp set review. Does not cover STR (Smith Travel Research) reporting, which is purchased separately and covers broader market data.

## Responsibilities

- **General Manager:** Conducts the weekly rate shop; interprets findings; makes pricing adjustments.
- **Ownership:** Reviews monthly comp set positioning as part of ownership check-in; approves any change to the defined comp set.

## The Comp Set

A comp set is the group of properties the GM uses as pricing reference. Salinas Point Inn's comp set consists of six properties. The comp set is deliberately tight — a larger comp set dilutes signal.

### Comp Set Selection Criteria

Comp set properties share these characteristics with Salinas Point Inn:

1. Similar property size (60–120 rooms).
2. Similar positioning (boutique/upscale independent).
3. Geographic overlap in the booking decision (a guest considering Salinas Point Inn might also consider these properties).
4. Similar amenity profile (breakfast included, no major brand flag).
5. Similar average room type distribution.

### Review Cadence

The comp set is reviewed twice a year — once in spring, once in fall. Changes to the comp set require ownership approval because historical trend data depends on consistent membership.

## The Weekly Rate Shop

Performed every Tuesday morning. Tuesday is chosen because weekend rate updates have settled and the data represents the week's actual positioning.

### Step 1 — Define the Shop

Shop 7 dates:

1. Tonight
2. Friday of this week
3. Saturday of this week
4. 14 days out (Friday)
5. 14 days out (Saturday)
6. 30 days out (Saturday)
7. A peak date within the next 90 days (holiday, known event, high-demand period)

### Step 2 — Gather Data

For each shopped date, capture the rate displayed for each comp set property.

1. Search each property on the same channel. Booking.com is the default channel for shops because rates are typically accurate and displayed without login.
2. Use identical search parameters: 1 night, 2 adults, standard room type (king or queen equivalent).
3. Record the total rate displayed to the guest (not the "from" rate, which can be manipulative).
4. Record the rate in a standardized units — if one property displays pre-tax and another displays post-tax, normalize to pre-tax.

### Step 3 — Build the Comp Shop Report

Build a table in the Comp Shop Log spreadsheet.

| Date | SPI | Comp 1 | Comp 2 | Comp 3 | Comp 4 | Comp 5 | Comp 6 | Avg (excl SPI) | SPI vs Avg |
|---|---|---|---|---|---|---|---|---|---|

"SPI" is Salinas Point Inn. The average of the comp set provides the market midpoint. The property's position vs. the average is the signal.

### Step 4 — Interpret

Typical positions for a boutique/upscale independent:

- **Above the mid-point by 5–15%:** Premium positioning. Sustainable if amenity, condition, and review scores support it.
- **At the mid-point (within 5%):** Neutral positioning. The property is priced with the market.
- **Below the mid-point by 5–15%:** Discounted positioning. Sustainable if done deliberately (strategic low-rate window, event-specific promotion). Unsustainable if unintentional.
- **More than 15% above or below:** Material divergence. Requires investigation.

For each shopped date, note the position in the report.

### Step 5 — Identify Actions

From the shop report, identify two categories of dates:

#### Dates to Raise

- Property is below the mid-point by 10%+ and demand is healthy (pace ahead of prior year).
- Property is below the mid-point on a peak date that the market has priced up but the property hasn't.

#### Dates to Evaluate

- Property is above the mid-point by 10%+ and pace is soft — may be overpriced for current demand.
- Property is below the mid-point and has been for several consecutive shops — may reflect a pricing floor that no longer matches the market.

#### Dates to Hold

- Property is within 5% of the mid-point regardless of direction.
- Property is deliberately positioned (above for a premium event, below for a promotion period).

## Integration with Daily Rate Review

The Tuesday shop feeds directly into Tuesday's Daily Rate Review (see Daily Rate Review and Yield Decision Procedure SOP).

1. For each "Dates to Raise" entry, evaluate whether to raise immediately or monitor for another cycle.
2. For each "Dates to Evaluate" entry, investigate before making a change — pace softness may be property-specific rather than rate-related.
3. Document any rate change that results from the shop in the Yield Log with specific reference to the shop data.

## Trend Analysis

Single-week shops are snapshots. Trends emerge from consecutive weeks.

### Monthly Trend Review

On the first Monday of each month, review the prior month's four shops:

1. For each comp set property, note the rate trajectory over the month.
2. Identify comp set properties that have moved rates materially up or down.
3. Assess whether Salinas Point Inn's rate trajectory has matched the market.

### Quarterly Positioning Review

Each quarter, produce a positioning summary:

1. Average position vs. comp set mid-point for each month.
2. Deliberate positioning decisions made and why.
3. Changes to the property or market that affected positioning.
4. Recommendations for the next quarter.

Distributed to ownership as part of the quarterly review.

## Special Shops

Beyond the weekly cadence, additional shops are conducted for specific events.

### Event Shop

2–3 months before a major regional event (Pebble Beach Pro-Am, major festival, wedding season weekends), the GM runs an event shop covering that window:

1. Shop each day of the event period.
2. Compare comp set positioning specific to the event.
3. Decide the property's positioning — premium, neutral, or below — and document.

### Compression Shop

On days when the market is clearly selling up (known major event, weather-driven displaced demand from a neighbor), the GM performs a mid-day shop to catch comp set rate movements:

1. Run the shop at 10:00 AM, 2:00 PM, and 6:00 PM.
2. Adjust property rate in near-real-time to capture compression.
3. This is rare — most days do not warrant mid-day shops.

### Response Shop

When a comp set property appears to have made a major pricing move (significantly raised or lowered), perform a targeted shop on that property:

1. Verify the move is sustained across multiple dates.
2. Determine if it's a one-off promotion or a strategic repositioning.
3. Decide whether to respond, hold, or wait for more data.

## Not What a Rate Shop Answers

The rate shop answers "what is the market priced at?" It does not answer:

- "Am I selling too much or too little?" (That's pace analysis.)
- "Is my rate fair for my product?" (That's a strategic question requiring guest feedback and review data.)
- "Is the market going to rise or fall?" (That's forecasting.)
- "Should I worry about a specific competitor?" (That's competitive analysis beyond rate.)

The GM uses the shop as one input among many, not as a pricing autopilot.

## Shop Methodology Discipline

Consistency matters more than speed.

- Always shop on the same day of week (Tuesday).
- Always shop the same dates relative to today.
- Always use the same channel.
- Always use the same search parameters.
- Always capture the same rate type (pre-tax, room only, standard room).

Inconsistent methodology produces noisy data that can't be compared week-over-week.

## Common Failure Modes

- Shopping "at a convenient time" rather than a consistent time — rates move; same-time-each-week is the discipline.
- Shopping only the immediate weekend — misses the forward-looking picture.
- Treating a single week's shop as actionable without trend confirmation — one-off movements can be noise.
- Including a property in the comp set that isn't actually competitive (e.g., a luxury property that doesn't take the same bookings) — dilutes signal.
- Not documenting deliberate positioning — next month's review will flag "why are we 12% below the midpoint" when the answer is "we intentionally ran a three-week below-midpoint promotion."

## Documentation

- **Comp Shop Log** — weekly Tuesday shop entered as a dated tab in the Revenue drive spreadsheet.
- **Monthly Trend Summary** — one-page summary filed in the Revenue folder.
- **Quarterly Positioning Review** — distributed to ownership with the quarterly forecast.
- **Comp Set Definition** — the current comp set and selection rationale, filed in the Revenue folder.

## Escalation Triggers

Contact ownership if:
- Comp set positioning shows Salinas Point Inn as a chronic discount property (below mid-point for 3+ consecutive months without strategic intent)
- A comp set property makes a major pricing move that materially changes the market
- A proposed change to the comp set (adding or removing a property)
- An event or compression opportunity suggests a deliberate major pricing move that deviates from standard practice
